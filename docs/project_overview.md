# Документация проекта "Веб-приложение для записи на проекты"

## Постановка задачи
Разработать веб-приложение, которое позволит студентам объединяться в группы и удобно распределяться по проектам в рамках учебных курсов (например, ОПРПО, УПРПО, ВвНСУБД). Приложение должно автоматизировать процесс сбора предпочтений, распределения и визуализации результатов.

## Требования

### Функциональные требования
1. **Управление курсами**
   - Каждый курс – отдельная сущность (например, “ОПРПО 2026”).
   - Преподаватель может создавать, редактировать и удалять курсы.
2. **Авторизация и роли**
   - Студенты, преподаватели и администраторы входят через систему авторизации (JWT).
   - Роли определяют доступный функционал.
3. **Импорт данных**
   - Импорт списка студентов из CSV.
   - Импорт списка проектов из CSV.
4. **Настройка параметров распределения**
   - Количество студентов в группе, дедлайны, минимальные/максимальные размеры команд.
5. **Визуализация**
   - Дашборд с текущими результатами распределения (таблицы, графики).
6. **Автоматическое распределение**
   - После дедлайна система автоматически назначает оставшихся студентов на свободные места.
7. **Обмен проектами**
   - Возможность для команд обмениваться проектами (с подтверждением преподавателя).

### Нефункциональные требования
- Технологии: TS (бэкенд Node.js/NestJS, фронтенд Vue 3), MySQL, Docker.
- Контейнеризация: Docker Compose для развёртывания.
- API должно быть описано в OpenAPI (Swagger).
- Код должен быть покрыт тестами (unit, интеграционными).

## Сценарии использования
1. **Авторизация пользователя** – вход по email/паролю, получение роли.
2. **Преподаватель импортирует студентов** – загружает файл, система парсит и сохраняет.
3. **Студент просматривает доступные проекты** – видит список проектов курса.
4. **Студент выбирает проект** – отправляет заявку (до дедлайна), при условии полной группы.
5. **Автоматическое распределение** – запускается вручную или по крону, назначает нераспределённых.
6. **Команда инициирует обмен** – отправляет запрос другой команде, преподаватель подтверждает.
7. **Просмотр дашборда** – графики и таблицы с распределением.

## Запуск без **Docker**

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/moevm/mse1h2026-project
   ```
2. Перейдите в директорию проекта:
    ```bash
    cd mse1h2026-project
    ```
3. Установите зависимости:
    ```bash
    pnpm i
    ```
4. Запустите проект
   1. Dev:
        ```bash
        pnpm dev
        ```
    2. Production:
        ```bash
        pnpm build
        pnpm preview
        ```
    3. Для запуска по отдельности фронтенда и бэкенда, используйте команды:

        `b` - бэкенд, `f` - фронтенд
        
        Dev:
        ```bash
        pnpm b dev
        pnpm f dev
        ```

        Production:
        ```bash
        pnpm b build
        pnpm b start
        pnpm f build
        pnpm f preview
        ```
    
    На данный момент фронтенд доступен по адресу `http://localhost:5173`, а бэкенд - `http://localhost:3000`.
