# mse-template

## Сценарии использования
 
*Сценарий использования* - описание того, что делает пользователь с приложением для достижения своей цели.
 
Роли пользователей - студент, преподаватель, администратор.
 
### Студент

**Основной сценарий**: авторизация -> просмотр списка курсов -> просмотр деталей курса -> просмотр списка проектов -> формирование и отправление запроса на объединение в команду | получение запроса на объединение в команду -> отправление заявки на выбранные проекты -> уведомление о распределении проектов -> просмотр распределения студентов по проектам -> инициация обмена проектами | получение запроса на обмен проектами.
 
1. Авторизация -> вход в систему;

2. Просмотр списка курсов -> отображение актуальных курсов;

3. Просмотр деталей курса -> отображение полной информации о курсе (описание, дедлайны);

4. Просмотр списка проектов -> отображение всех проектов, на которые можно записаться;

  - 5.1 Формирование и отправление запроса на объединение в команду -> уведомление студентам о запросе;
  - 5.2 Получение запроса на объединение в команду -> согласие на объединение | отказ;

6. Отправление заявки на выбранные проекты -> заявка сохраняется в системе;

7. Уведомление о распределении проектов <- система отправляет уведомление об окончании распределения (после запуска алгоритма администратором);

8. Просмотр распределения студентов по проектам (дашборд) -> просмотр сформированных команд, назначенных на проекты;

  - 9.1. Инициация обмена проектами -> отправление уведомления участникам другой команды и преподавателю;
  - 9.2. Получение запроса на обмен проектами -> подтверждение обмена | отказ от обмен.

### Преподаватель

**Основной сценарий**: авторизация -> просмотр списка курсов -> просмотр деталей курса -> настройка содержания курса -> просмотр списка проектов -> просмотр распределения студентов по проектам -> получение запроса на подтверждение обмена проектами.

1. Авторизация -> вход в систему;

2. Просмотр списка курсов -> отображение актуальных курсов;

3. Просмотр деталей курса -> отображение полной информации о курсе (описание, дедлайны);

4. Настройка содержания курса (редактирование описания, дедлайнов, списка проектов) -> изменение параметров курса, курс готов к работе;

5. Просмотр списка проектов -> отображение всех существующих проектов;

6. Просмотр распределения студентов по проектам (дашборд) -> просмотр сформированных команд, назначенных на проекты;

7. Получение запроса на подтверждение обмена проектами -> подтверждение обмена | запрет обмена.

### Администратор

**Основной сценарий**: авторизация -> просмотр списка курсов -> настройка параметров распределения -> запуск процесса распределения -> визуализация текущих результатов распределения -> корректировка результатов -> публикация результатов распределения.

1. Авторизация -> вход в систему;

2. Просмотр списка курсов -> отображение актуальных курсов;

3. Настройка параметров распределения (сколько студентов, сроки, алгоритм) -> параметры сохранены, кнопка запуска распределения становится активной;

4. Запуск процесса распределения -> система выполняет алгоритм, проекты закрепляются за командами;

5. Визуализация текущих результатов распределения -> отображены дашборды по созданной реализации (все ли распределились, есть ли пустые проекты);

6. Корректировка результатов -> получение итогового оптимального распределения;

7. Публикация результатов распределения -> студенты и преподаватели получают доступ к дашбордам.

## Установка и запуск

### Запуск без **Docker**

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/moevm/mse1h2026-project
   ```
2. Перейдите в директорию проекта:
    ```bash
    cd mse1h2026-project
    ```
3. Установите зависимости:
    ```bash
    pnpm i
    ```
4. Запустите проект
   1. Dev:
        ```bash
        pnpm dev
        ```
    2. Production:
        ```bash
        pnpm build
        pnpm preview
        ```
    3. Для запуска по отдельности фронтенда и бэкенда, используйте команды:

        `b` - бэкенд, `f` - фронтенд
        
        Dev:
        ```bash
        pnpm b dev
        pnpm f dev
        ```

        Production:
        ```bash
        pnpm b build
        pnpm b start
        pnpm f build
        pnpm f preview
        ```
    
    На данный момент фронтенд доступен по адресу `http://localhost:5173`, а бэкенд - `http://localhost:3000`.
